     1→# NIP — Consolidated Logic & Function Testing Engine
     2→## Master Test Case Catalog
     3→### Every Law (L1–L14), Module (M1–M9), and Room (0–4 + 0.5) from the full spec set, expressed as runnable test cases
     4→
     5→**How to use this document:**
     6→- Each test has an ID (`L1-01`, `M4-03`, `R2-05`, etc.) — reference these in bug reports and rectification notes.
     7→- Each test states: **Setup** (what data/state to create), **Action** (what to run), **Expected** (what must be true), and where applicable a **Sabotage variant** (the deliberately broken input that must be caught, per L9 — "demonstrated, not built").
     8→- Status column: mark ✅ PASS / ❌ FAIL / ⚠️ PARTIAL / ⬜ NOT RUN as you execute.
     9→- A companion file, `nip_test_suite_scaffold.ts`, implements the highest-priority tests below as runnable Jest/Vitest code — drop it into `tests/` and adapt the import paths to your repo.
    10→- Priority tags: **P0** = safety-critical, blocks everything; **P1** = core correctness; **P2** = quality/completeness.
